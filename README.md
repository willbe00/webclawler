# Лабораторная работа — HTTP клиент и сервер

**Задание:** реализовать HTTP клиент и сервер для сбора информации из интернета.

---

## Что сделано

Написал сервер на Node.js + Express и клиент на чистом JS.

Сервер хранит список ключевых слов, каждому из которых соответствует несколько URL. Клиент отправляет ключевое слово — сервер отдаёт список ссылок. Затем клиент выбирает нужную и через сервер скачивает контент. Сервер при этом передаёт размер файла и прогресс загрузки. Всё что скачали — сохраняется в localStorage и доступно офлайн.

---

## Запуск

Нужен Node.js 18+.

```bash
npm install
npm start
```

Открыть в браузере: http://localhost:3000

## Как устроен сервер

В `server/index.js` есть объект `keywordDatabase` — это и есть база ключевых слов:

```js
const keywordDatabase = {
  javascript: [
    { url: 'https://...', label: 'Express README' },
    { url: 'https://...', label: 'GitHub Info' },
  ],
  python: [ ... ],
};
```

Три эндпоинта:

- `GET /api/keywords` — отдаёт все ключевые слова
- `GET /api/urls?keyword=nodejs` — отдаёт список URL для слова
- `GET /api/download?url=...` — скачивает контент и стримит его клиенту

При скачивании сервер использует Axios в режиме stream и пишет в заголовки `X-Content-Length` (общий размер) и `X-Downloaded-Bytes` (сколько уже скачано). Клиент читает эти заголовки и обновляет прогресс-бар.

Также есть whitelist — скачать можно только URL из базы, произвольные адреса сервер не пропустит.

## Как устроен клиент

Всё в одном файле `public/index.html`.

Запросы к серверу идут через `fetch()` без перезагрузки страницы. Прогресс загрузки отслеживается через `ReadableStream` — данные читаются кусками и считается сколько байт пришло:

```js
const reader = response.body.getReader();
while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  received += value.length;
  // обновляем прогресс-бар
}
```

Скачанный контент сохраняется в `localStorage`. Если JSON — форматируется и подсвечивается. Все ошибки показываются пользователю прямо в интерфейсе.
